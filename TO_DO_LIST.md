## 問題一：兩個主方案的大量規則重複內嵌，未善用 lib

sgi_ping_im.schema.yaml 的 speller.algebra、preedit_format、comment_format
超過 600 行，全部內嵌在 schema 內，與 lib_sip_ngoo_im.yaml 中的
phah_ji_tng_uann 區塊邏輯高度重疊。

sgi_zu_im 的 comment_format 則有使用 \_\_patch 引用外部 lib，但路徑名稱
tlpa_kik_kan_sip_goo_im_libs:/hau_suann_tuann 與實際存在的
lib_hau_suan_ji_tuann.yaml 的 key hau_suan_tuann
並不能直接對應，需確認此路徑是否有效。

建議：將 sgi_ping_im 的 preedit_format 和 comment_format 也改為透過 \_\_patch
引用 lib，實現真正的模組化。

## 問題三：rime.lua 的聲韻母字典與 YAML preedit 規則各自維護

rime.lua 中定義了：
local sheng_mu_dict = { ["邊"]="p", ["頗"]="ph", ... }
local yun_mu_dict = { ["君"]={"un","ut"}, ["堅"]={"ian","iat"}, ... }

lib_sip_ngoo_im.yaml 的 phah_ji_tng_uann 也有同樣的聲韻母映射（以 xform
規則形式）。兩者各自獨立，若新增或修改聲韻母，必須同時修改兩處，容易遺漏。

建議：短期可在兩處都加上對照表位置的交叉註解；長期可考慮將字典資料單獨整理為一
份文件（如 docs/聲韻母對照表.md）作為唯一來源。

## 問題四：sgi_ping_im 的零聲母識別不完整

# sgi_ping_im schema 第 214 行

- xform/^([aeiou])/英$1/

這只涵蓋 a e i o u 開頭，但台語有 oo
韻（如「烏」oo1、「翁」ong1）、以及韻化聲母
m（如「毋」m7）、ng（如「硬」ng7）。sgi_zu_im 的 preedit
則有更完整的處理（明確列出 ㆦ、ㆬ、ㆭ 等）。

建議：在 sgi_ping_im 的零聲母規則中補上 oo 開頭及韻化聲母的識別：

- xform/^([aeiou])/英$1/
- xform/^(oo)/英$1/ # 補上 oo 韻
- xform/^m(\d)/英姆$1/ # 韻化聲母 m
- xform/^ng(\d)/英鋼$1/ # 韻化聲母 ng

● 改動完成。以下是所做的修改說明：

修改內容（sgi_ping_im.schema.yaml）

1. 新增韻化聲母直接識別（lines 215-218）

- xform/^m(一|二|三|五|七|四|八)/英姆$1/ # 韻化聲母 m（如：毋 m7）
- xform/^ng(一|二|三|五|七|四|八)/英鋼$1/ # 韻化聲母 ng（如：硬 ng7）

放在聲母映射規則之前，讓 m七/ng七 直接走零聲母路徑，不再繞道
毛→英姆。涵蓋了原本遺漏的入聲調 四八。

2. 補上 oo 韻明確識別（line 220）

- xform/^oo/英oo/ # 補上 oo 韻（如：烏 oo1、翁 ong1）

放在 ^([aeiou]) 之前，確保 oo 以整體被識別（原本 o 雖在 [aeiou]
內，但僅匹配首字元，結果雖相同，行為上不夠明確）。

3. 備援規則補強調號範圍（lines 388-389）

- xform/毛(一|二|三|五|七|四|八)/英姆$1/ # 原本只有 一二三五七
- xform/雅(一|二|三|五|七|四|八)/英鋼$1/ # 同上

補上入聲調 四八，並加上說明這是備援規則。

## 問題五：字典主檔的 name 與檔名不對應

tl_han_ji_khoo.dict.yaml 的 header：
name: tl_ji_khoo_peh_ue ← 與檔名 tl_han_ji_khoo 不同

RIME 以 name 為字典的識別 ID，而 schema 的 translator.dictionary:
tl_han_ji_khoo 是依檔名引用。兩者不一致雖然 RIME 目前可能仍能運作（RIME
優先以檔名匹配），但在其他方案引用此字典時容易造成混淆。

建議：將 name 改為 tl_han_ji_khoo 與檔名保持一致；或反過來重命名檔案為
tl_ji_khoo_peh_ue.dict.yaml，並更新所有 schema 的引用。

```
# Rime dictionary
# encoding: utf-8
#
# 河洛白話音
#
---
name: tl_han_ji_khoo
version: "v0.1.0.0"
sort: by_weight
use_preset_vocabulary: false
columns:
  - text    # 漢字
  - code    # 台灣音標（TLPA）拼音
  - weight  # 常用度（優先顯示度）
  - stem    # 用法舉例
  - create  # 建立時間
import_tables:
  # - tl_ji_khoo_kah_kut_bun
...
㔂	lan5	0.1	NA	2026-01-02 22:50:38
㜅	thik4	0.01	NA	2025-02-11 11:16:04
```

這行是整個方案的鍵盤映射核心，但沒有分組或說明，任何一個字元的修改都必須同時數
